Unit

TelemetryRecipient

Declaration

type TTelemetryRecipient = class(TTelemetryVersionPrepareObject)

Description

TTelemetryRecipient is used as a main way to control the telemetry API.

It provides methods for events and channels registration, unregistration and many others. It also provides object events that wraps API event and channel callbacks.
Before instance of this class is created, a check if it supports required telemetry and game version must be performed. For that purpose, it has class methods that can be called on class itself (before creating class instance). If required versions are not supported, do not create instance of this class!

Note: Because recipient internally creates instance of TTelemetryInfoProvider, the provider must support required versions of telemetry and game as well.

These methods: 

    HighestSupportedTelemetryVersion
    SupportsTelemetryVersion
    SupportsTelemetryMajorVersion
    SupportsGameVersion
    SupportsTelemetryAndGameVersion

...can and actually must be called directly on class. They should be used to check whether both this class and TTelemetryInfoProvider class supports required telemetry and game version before instantiation (creation of class instance).

procedure ProcessConfigurationEvent(const Data: scs_telemetry_configuration_t); virtual;

Method used for processing configuration events. It is called from EventHandler method when configuration event is received and StoreConfigurations is set to True.
Received data are parsed and configuration informations are extracted and stored in StoredConfigs list. OnConfig event is called for every extracted config (after it is stored / its stored value changed).
When ManageIndexedChannels is set to True, then indexed channels are too managed in this method (refer to source code for details).

Parameters
Data
Pointer to structure holding actual configuration data. Must not be Nil.

Function ChannelRegisterAll(RegPrimaryTypes: Boolean = True; RegSecondaryTypes: Boolean = False; RegTertiaryTypes: Boolean = False): Integer; virtual;

This method registers all channels that the TTelemetryInfoProvider class is aware of for current telemetry and game version.
When channel is marked as indexed, then all channel and index combinations are registered (see implementation of this method for details), otherwise only channels with index set to SCS_U32_NIL are registered. When ManageIndexedChannels and StoreConfigurations properties are both set to True, then top index for indexed channel registration is taken from appropriate stored configuration value.
Set RegPrimaryTypes, RegSecondaryTypes and RegTertiaryTypes to true to register respective channel value type. If all three parameters are set to false, the method executes but nothing is registered.

property KeepUtilityEvents: Boolean read fKeepUtilityEvents write SetKeepUtilityEvents;

When set to True, the recipient automatically registers all known events marked as utility and refuses to unregister such events.
This property is intended to ensure the recipient will stay responsive, because events and channels can be (un)registered only in events callbacks. If no event would be registered, then such call will not occur, rendering recipient unresponsive.
Initialized to True.

property StoreConfigurations: Boolean read fStoreConfigurations write SetStoreConfigurations;

If True, any configuration data passed from the game are parsed and stored.
When set to True, the configuration event is automatically registered.
When set to False, StoredConfigs list is cleared.
Initialized to True.

property ManageIndexedChannels: Boolean read fManageIndexedChannels write fManageIndexedChannels;

When True, registration and unregistration of indexed channels that are index-binded to some configuration is automatically managed.
Affected methods:

  • ChannelRegisterAll - binded channels are registered up to index (count - 1) stored in appropriate config. If such config is not stored, they are registered in the same way as not binded channels.

  • ProcessConfigurationEvent - when binded config is parsed, all registered channels are scanned and those binded to this config are proccessed in following way:
    New config value is greater than old value: channels with indices above new config value are unregistered.
    New config value is smaller than old value: when channels with all indices from old config value are registered, then new channels with indices up to the new value are registered.

Initialized to False.

property StoreChannelValues: Boolean read fStoreChannelValues write SetStoreChannelValues;

When True, all incoming channels along with their values are stored in StoredChannelValues list.
When set to False, StoreChannelValues list is cleared.
Initialized to False.
