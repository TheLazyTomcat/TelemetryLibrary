Declaration

type TAddressList = class(TStringList)

Description

List used to store addresses that can or cannot, depending on list mode, connect to the server.

Addresses are stored as strings. All comparisons are case insensitive.

Function ClientCanConnect(Address: String): Boolean; virtual;

Use this function to check, whether client on given address can or cannot connect.
Result of this function depends on ListMode property value. When ListMode is set to invalid/unknown value, this function returns False.

Parameters
Address
Address of connecting client as string.
Returns

True when client is allowed to connect, otherwise False.

Function ClientIsInList(Address: String): Boolean; virtual;

Indicates whether given address is listed (case insensitive).

Parameters
Address
Address to be checked in string format.
Returns

True when address if found in list, otherwise False.

procedure SaveToFile(const FileName: String = ''); override;

Saves entire list to text file (one address per line).
When "FileName" parameter is set to an empty string, then the name of file to which the list is saved is taken from FileName property.

Parameters
FileName
Name of file (fully qualified path) to which the list will be saved.
procedure LoadFromFile(const FileName: String = ''); override;

Clears the list and loads new content from text file.
When "FileName" parameter is set to an empty string, then the name of file from which the list is loaded is taken from FileName property.

Parameters
FileName
Name of file (fully qualified path) from which the list will be loaded.

Properties

property ListMode: TAddressListMode read fListMode write fListMode;

Property determining how the list will evaluate whether client can ot cannot connect (see ClientCanConnect method for details). For available values and behaviour for each of them, refer to TAddressListMode enumerated type.
Initialized to lmBlackList.

property FileName: String read fFileName write fFileName;

File name used in SaveToFile and LoadFromFile methods. Refer to them for details.
Initialized in following way: 

    ExtractFilePath(GetModuleName(hInstance)) + def_AddressListFileName;

...where def_AddressListFileName is string constant of value "address.list.txt" (without quotes). It means the file would be located in the same folder as executable or library compiled with this unit.
