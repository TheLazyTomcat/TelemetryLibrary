Declaration

type TCircularItemsBuffer = class(TObject)

Description

Ancestor class for other circular buffer classes.

It is implemented as one way circular buffer, meaning it is created with fixed size, and is used where there is need for buffer that prevents uncontrolled growth of memory use. When the buffer becames full, newly added items simply overwrites the oldest ones (they are dropped).
Buffer internally uses TCircularBufferItem structures to store individual buffered items, but only field Data of this structure can be accessed by user. So when adding, peeking, removing, etc. any item, only this pointer is accessed, everything else is managed automatically.

procedure AddItem(Item: Pointer); virtual;

Use this method to add new item to the buffer.
If buffer is full, then item at current position is dropped (DropItem method is called) and replaced by the passed one.
Item parameter must not be nil, otherwise nothing is added.
OnItemAdd event is called after successful addition.

Pointer that should be added to buffer. Must not be Nil.
