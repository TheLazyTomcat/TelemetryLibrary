Description

Provides types, constants and routines used for manipulation with value types.

Last change: 2016-03-20

Each telemetry channel has a primary type, which is a native type for such channel. But telemetry is also able to provide value of some channels as a secondary type when registered so, eg. as 64bit integer for a channel whose native type is a 32bit integer.
Following is table of secondary types for each existing primary type ("/" means this primary type does not allow any secondary type to be used): 

    Primary type             -    Secondary types
 -------------------------------------------------------------------
  SCS_VALUE_TYPE_bool        -  /
  SCS_VALUE_TYPE_s32         -  /
  SCS_VALUE_TYPE_u32         -  u64
  SCS_VALUE_TYPE_u64         -  /
  SCS_VALUE_TYPE_float       -  double
  SCS_VALUE_TYPE_double      -  float
  SCS_VALUE_TYPE_fvector     -  dvector
  SCS_VALUE_TYPE_dvector     -  fvector
  SCS_VALUE_TYPE_euler       -  /
  SCS_VALUE_TYPE_fplacement  -  dplacement, fvector, dvector, euler
  SCS_VALUE_TYPE_dplacement  -  fplacement, dvector, fvector, euler
  SCS_VALUE_TYPE_string      -  /

This unit is designed to provide any material (functions, types, ...) needed for identification and registration of secondary types for any telemetry channel.

Uses

  • TelemetryStrings

procedure CompressValueTypesArray(var ValueTypes: TValueTypesArray);

Compresses value types array, that is, it moves all valid value types to the start of the array so there are no invalid types between valid ones.

For example, following array: 

  s32, s64, invalid, float, invalid, invalid, double

will look like this after the compression: 

  s32, s64, float, double

Parameters
ValueTypes
Array that has to be comressed.

Function BitmaskValueType(Bitmask: TValueTypeBitmask): scs_value_type_t;

Returns value type corresponding to first set bit (counting from bit 0 up) in the Bitmask parameter.

If no bit is set, SCS_VALUE_TYPE_INVALID will be returned.
For example, for bitmask 0x0000001A (...00011010), SCS_VALUE_TYPE_s32 will be returned as it corresponds to bit 1.
To obtain all selected types, use function BitmaskValueTypes.

Parameters
Bitmask
Bitmask containing selected type(s).
Returns

First selected value type from the bitmask.

Function BitmaskValueTypesAddPrimary(Bitmask: TValueTypeBitmask; PrimaryValueType: scs_value_type_t): TValueTypesArray;

Returns an array of all value types selected in the passed bitmask including passed primary value type.

Primary value type will be always in the first item, which also means the resulting array will newer be empty if you pass valid primary type. If you pass an invalid type as primary, the array will be empty as the first item will be set to invalid.
For Example, for bitmask 0x00000009 (...00001101) and primary type of SCS_VALUE_TYPE_double, following array will be returned: 

  double, bool, u32, u64

Function SelectSupportedValueTypes(PrimaryValueType: scs_value_type_t; SecondaryValueTypesBitmask: TValueTypeBitmask; SelectPrimary: Boolean; SecondarySelectionMask: UInt32): TValueTypesArray; overload;

Returns an array of value types supported by a selected primary type according to a selection mask.

Array is constructed in the following way:

  • The array is emptied (all items are set to SCS_VALUE_TYPE_INVALID)

  • If SelectPrimary is True, then first item is set to a passed primary value type

  • Other items are filled from parameter SecondaryValueTypesBitmask

  • All items in the array (excluding the primary if it is selected) are checked against corresponding bit in the selection mask, when the bit is not set, they are removed from the array

  • The array is compressed

Let's explain more it on some examples:

  • Example 1

    PrimaryValueType = SCS_VALUE_TYPE_fplacement
    SecondaryValueTypesBitmask = 0x000005C0 (...0000010111000000)
    SelectPrimary = True
    SecondarySelectionMask = 0x00000005 (...00000101)

    Intermediate array will look like so: 

      fplacement, fvector, dvector, euler, dplacement

    Then a selection mask is applied (remember that primary type is selected and it cannot be removed, so the selection mask is applied from second item) with this result: 

      fplacement, fvector, invalid, euler, invalid

    And finally, after compression, the result is: 

      fplacement, fvector, euler

  • Example 2

    PrimaryValueType = SCS_VALUE_TYPE_dplacement
    SecondaryValueTypesBitmask = 0x000002C0 (...0000001011000000)
    SelectPrimary = False
    SecondarySelectionMask = 0x00000001 (...0000001)

    Intermediate array: 

      fvector, dvector, dplacement

    After selection mask is applied, final result is: 

      fvector

Types

TValueTypeBitmask = UInt32;

Bitmask type used to identify any number of value types.

Each bit in this bitmask corresponds to exactly one known value type. When a particular bit is set, it means such type is used or selected. Currently, individual bits are mapped as follows: 

    bit 0   - SCS_VALUE_TYPE_bool
    bit 1   - SCS_VALUE_TYPE_s32
    bit 2   - SCS_VALUE_TYPE_u32
    bit 3   - SCS_VALUE_TYPE_u64
    bit 4   - SCS_VALUE_TYPE_float
    bit 5   - SCS_VALUE_TYPE_double
    bit 6   - SCS_VALUE_TYPE_fvector
    bit 7   - SCS_VALUE_TYPE_dvector
    bit 8   - SCS_VALUE_TYPE_euler
    bit 9   - SCS_VALUE_TYPE_fplacement
    bit 10  - SCS_VALUE_TYPE_dplacement
    bit 11  - SCS_VALUE_TYPE_string
    bit 12+ - unused

Note that SCS_VALUE_TYPE_INVALID type is not mapped, hence you cannot use or select it.

TValueTypesArray = Array[0..SizeOf(TValueTypeBitmask) * 8] of scs_value_type_t;

An array type used in the same manner as TValueTypeBitmask, but individual types does not have its hardcoded placement.

All valid items (types) must be placed at the start of the array, and first item that contains an SCS_VALUE_TYPE_INVALID type marks the end of the array. If the first item in this array is invalid, then whole array should be considered empty.
