Unit TelemetryRecipient

DescriptionUsesClasses, Interfaces, Objects and RecordsFunctions and ProceduresTypesConstantsVariables

Description

Telemetry recipient class (API control and data receiver).

TelemetryRecipient

©František Milt, all rights reserved.

This unit contains TTelemetryRecipient class used to control the telmetry API (see class declaration for details) and few classes used to manage multicast events for the recipient, namely: 

  TMulticastLogEvent
  TMulticastEventRegisterEvent
  TMulticastEventEvent
  TMulticastChannelRegisterEvent
  TMulticastChannelUnregisterEvent
  TMulticastChannelEvent
  TMulticastConfigEvent

Included files:

    .\Inc\TelemetryRecipient_MulticastEvents.pas
      Contains declarations and implementations of multicast event classes.

Overview

Classes, Interfaces, Objects and Records

Name Description
Class TMulticastLogEvent Class used to manage multicast event called when recipient writes to game log.
Class TMulticastEventRegisterEvent Class used to manage multicast event called when telemetry event is registered or unregistered.
Class TMulticastEventEvent Class used to manage multicast event called when telemetry when telemetery event occurs.
Class TMulticastChannelRegisterEvent Class used to manage multicast event called when telemetry channel is registered.
Class TMulticastChannelUnregisterEvent Class used to manage multicast event called when telemetry channel is unregistered.
Class TMulticastChannelEvent Class used to manage multicast event called when telemetery channel callback occurs.
Class TMulticastConfigEvent Class used to manage multicast event called when config is parsed from configuration telemetry event.
Class TTelemetryRecipient TTelemetryRecipient is used as a main way to control the telemetry API.

UserData passed to streaming function along with this callback must contain valid TTelemetryRecipient object.

Recipient
