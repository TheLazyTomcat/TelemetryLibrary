Declaration

type TCustomTelemetryList = class(TObject)

Description

Ancestor class for all other list classes in TelemetryLists unit.

TCustomTelemetryList serves as ancestor for all other list classes declared in this unit. It wrapps around TList class and adds methods used for easier list management in descendant classes.

Description

Fields

fMainList: TList;

Internal list used to hold actual items. It is not publicly visible and is therefore managed automatically.

fUpdating: Boolean;

When this variable is set to True, it indicates that the list is updated from outside and OnChange event should not be called.
It is set by methods BeginUpdate and EndUpdate.

fOnChange: TNotifyEvent;

Holds reference to OnChange event handler.

fOnChangeMulti: TMulticastNotifyEvent;

Object handling multicast OnChange event.

procedure PtrInsert(Index: Integer; Item: Pointer); virtual;

Inserts new item at position given by Index. List Count is increased by one and all existing items from given position (included) up are moved higher. Item can be Nil. When index falls out of allowed boundary (<0,Count> - passed index can be higher than current highest index, if so, item is added at the end of the list), and exception is raised.
Calls method DoChange when item is successfully inserted.

Parameters
Index
Index at which the new item should be added.
Item
Inserted item.
Function PtrRemove(Item: Pointer): Integer; virtual;

Removes item passed in Item parameter. Item should not be, but can be Nil. When given item is not found in list, nothing is removed and the function fails with no error (-1 is returned). If the list contains more than one copy of the removed item, only the first occurence is removed.
Calls method DoChange when item is successfully removed.

Parameters
Item
Item that has to be removed.
Returns

Index at which the removed item was place before removal, -1 whem given item was not found in the list.

procedure BeginUpdate; virtual;

Sets fUpdating to True.

procedure EndUpdate; virtual;

Sets fUpdating to False and calls DoChange method.

procedure DoChange; virtual;

Calls handler of OnChange event (if assigned) - but only if fUpdating is set to False, otherwise the handler is not called and method returns immediately.

procedure Clear; virtual;

Deletes all items in the list. It only deletes stored pointers, allocated memory they are pointing to is not freed. Override this method in descendants to free memory used by items (put inherited code at the end).
Calls method DoChange after all items are deleted.

Properties

property Count: Integer read GetCount;

Number of items stored in the list.

property OnChange: TNotifyEvent read fOnChange write fOnChange;

Event called whenever the list is changed. It can also be called manually by calling DoChange method. It is NOT called when fUpdating is set to True (but can be still called directly if handler is assigned).

property OnChangeMulti: TMulticastNotifyEvent read fOnChangeMulti;

Multicast event called whenever the list is changed. It can also be called manually by calling DoChange method. It is NOT called when fUpdating is set to True (but can be still called directly).
Assigning handlers to this event does not interfere with normal OnChange event.
Normal OnChange event is called first, handlers of this event are called afterwards.
