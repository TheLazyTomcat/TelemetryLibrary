Description

Class designed to imitate behavior of telemetry_position example distributed with the Telemetry SDK.

It writes data to a textual log file whe same way as mentioned example, so parsers intended to read such log should have no problem reading log produced by this class.
Note that there may be slight behavioral differences as a consequence of use of different language and programming style (object instead of functions), but they should not pose any problem.

It writes world-space position of driver's head on per-frame basis to the output.
Resulting file can then look like this (<...> means a part of the text is removed): 

  Log opened
  Game 'eut2' 1.12
  WARNING: Head position unavailable
  -59601.889437;51.311265;-7795.376735
  -59601.889422;51.308747;-7795.376725
  -59601.890405;51.312907;-7795.378027

  -59601.888888;51.322672;-7795.375585
  -59601.887792;51.327103;-7795.373351

  <...>

  -59734.282434;51.712648;-7890.447486
  -59734.282899;51.712515;-7890.447426
  -59734.283343;51.712392;-7890.447367
  Log ended

constructor Create(Recipient: TTelemetryRecipient; const LogFileName: String = def_LogFileName);

Class constructor.

Manages everything what is necesary at the start of logging (clears fields, initializes log, registers events and channels, creates internal objects, ...).
Recipient parameter must be assigned, otherwise an exception is raised. LogFileName must not be empty.

Parameters
Recipient
Must contain valid reference to a TTelemetryRecipient instance.
LogFileName
Name of the output log file. File is stored in the same folder where a module containing this code is placed.
Exceptions raised
ETLNilReference

ETLInitFailed
