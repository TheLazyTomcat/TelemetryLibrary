Description

Class designed to imitate behavior of telemetry example distributed with the Telemetry SDK.

It writes data to a textual log file the same way as mentioned example, so parsers intended to read such log should have no problem reading log produced by this class.
Note that there may be slight behavioral differences as a consequence of use of different language and programming style (object instead of functions), but they should not pose any problem.

It writes complete configurations data when they are received and also following values on per-frame basis:

  • Timestamp [µs]

  • Raw rendering timestamp [µs]

  • Raw simulation timestamp [µs]

  • Raw paused simulation timestamp [µs]

  • Vehicle heading [degrees]

  • Vehicle pitch [degrees]

  • Vehicle roll [degrees]

  • Speed of the vehicle [m/s]

  • Engine revolutions [min-1]

  • Engaged gead

fFormatSettings: TFormatSettings;

Holds format settings used when converting floating point values to text.

Object doing actual write into the output file.
Managed internally.

fLogFileName: String;

Name of the output log file.

fOutputPaused: Boolean;

When True, output is paused and nothing is written to it (eg. when game is paused).

fPrintHeader: Boolean;

When True, a header should be written into the output in next SCS_TELEMETRY_EVENT_frame_end event.

fLastTimestamp: scs_timestamp_t;

Timestamp of the last frame (SCS_TELEMETRY_EVENT_frame_start event).
Initialized to -1.

constructor Create(Recipient: TTelemetryRecipient; const LogFileName: String = def_LogFileName);

Class constructor.

Manages everything what is necesary at the start of logging (clears fields, initializes log, registers events and channels, creates internal objects, ...).
Recipient parameter must be assigned, otherwise an exception is raised. LogFileName must not be empty.

Parameters
Recipient
Must contain valid reference to a TTelemetryRecipient instance.
LogFileName
Name of the output log file. File is stored in the same folder where a module containing this code is placed.
Exceptions raised
ETLNilReference
When aRecipient is not assigned.
ETLInitFailed
When log initialization fails (when method InitLog returns False).
ETLRegFailed
When registration of any of the following telemetry events fails: 

    SCS_TELEMETRY_EVENT_frame_start
    SCS_TELEMETRY_EVENT_frame_end
    SCS_TELEMETRY_EVENT_paused
    SCS_TELEMETRY_EVENT_started
