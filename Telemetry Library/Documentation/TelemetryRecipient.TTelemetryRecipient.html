Unit

TelemetryRecipient

Declaration

type TTelemetryRecipient = class(TTelemetryVersionPrepareObject)

Description

TTelemetryRecipient is used as a main way to control the telemetry API.

It provides methods for events and channels registration, unregistration and many others. It also provides object events that wraps API event and channel callbacks.
Before instance of this class is created, a check if it supports required telemetry and game version must be performed. For that purpose, it has class methods that can be called on class itself (before creating class instance). If required versions are not supported, do not create instance of this class!

Note - Because recipient internally creates instance of TTelemetryInfoProvider, the provider must support required versions of telemetry and game as well.

Recipient can, similarly to TTelemetryInfoProvider, be crated on two ways - used-manager and automatically managed. When it is created in user-managed mode, internal TelemetryInfoProvider is too created as user-managed and vice-versa. Here are main diferences between those two modes:

Automatically managed (created using any parametrized constructor):

  • TelemetryInfoProvider is created as automatically managed (parameters are passed from constructor) - list are automatically filled

  • Telemetry and game version-specific preparations are performed

  • Game callbacks are automatically assigned (when supported by used constructor)

  • KeepUtilityEvents property is set to True - utility events are automatically registered in constructor when it is possible (AllowAutoRegistration must be True and game callbacks must be already assigned; Note - only one constructor assigns them)

  • StoreConfigurations property is set to True - configuration event is automatically registered in constructor when it is possible

  • AllowAutoRegistration property is set to True (value of this property can be changed later)

  • (Un)Registration methods check whether requested item is already registered or not before actual (un)registration

User-manager mode (created using no-parameter constructor):

  • TelemetryInfoProvider is created as user-managed

  • No telemetry or game version-specific preparations are performed

  • Game callbacks are NOT automatically assigned - therefore things like event registration or writing to a game log does not work

  • KeepUtilityEvents property is set to False - utility events are NOT automatically registered in constructor

  • StoreConfigurations property is set to False - configuration event is NOT automatically registered in constructor

  • AllowAutoRegistration property is set to False (value of this property can be changed later)

  • (Un)Registration methods does NOT check whether requested item is already registered or not before actual (un)registration

These methods: 

    HighestSupportedTelemetryVersion
    SupportsTelemetryVersion
    SupportsTelemetryMajorVersion
    SupportsGameVersion
    SupportsTelemetryAndGameVersion

...can and actually must be called directly on class. They should be used to check whether both this class and TTelemetryInfoProvider class supports required telemetry and game version before instantiation (creation of class instance).

procedure ProcessConfigurationEvent(const Data: scs_telemetry_configuration_t); virtual;

Method used for processing configuration events. It is called from EventHandler method when configuration event is received.
Received data are parsed and configuration informations are extracted and stored in StoredConfigs list (StoreConfigurations must be True). OnConfig event is called for every extracted config (after it is stored / its stored value changed).
When ManageIndexedChannels is set to True, then indexed channels are too managed in this method (refer to source code for details).

Parameters
Data
Structure holding actual configuration data.

constructor CreateCurrent(GameID: TelemetryString; GameName: TelemetryString = '');

Specialized object constructor.

This constructor is designed to create an automatically managed instance prepared for latest supported version of passed game. It calls parametrized constructor with parameter TelemetryVersion set to value returned by function HighestSupportedTelemetryVersion, GameID set to passed game id and GameVersion set to value returned by function HighestSupportedGameVersion. Does not assign callbacks since they are not passed in paramters - you have to assign them manually after object creation.

Parameters
GameID
Game identifier.
GameName
Name of the game (optional parameter). This parameter is not procesed, only passed as is.

Registers specific telemetry event.
Works only when cbRegisterEvent callback is assigned, when it is not, function returns false and LastTelemetryResult is set to SCS_RESULT_generic_error. When instance was created as automatically managed, a check whether requested event is already registered is performed. When it is, function returns True but LastTelemetryResult is set to SCS_RESULT_already_registered. User-managed instance does not perform this check.

Parameters

Returns

When set to True, the recipient automatically registers all known events marked as utility (property AllowAutoRegistration must be True for this) and also refuses to unregister such events.
This property is intended to ensure the recipient will stay responsive, because events and channels can be (un)registered only in events callbacks. If no event would be registered, then such call will not occur, rendering recipient unresponsive.

Initialized to True.

property ManageIndexedChannels: Boolean read fManageIndexedChannels write fManageIndexedChannels;

When True, registration and unregistration of indexed channels that are index-binded to some configuration is automatically managed.
Affected methods:

  • ChannelRegisterAll - binded channels are registered up to index (count - 1) stored in appropriate config. If such config is not stored, they are registered in the same way as not binded channels.

  • ProcessConfigurationEvent - when binded config is parsed, all registered channels are scanned and those binded to this config are proccessed in following way:
    New config value is greater than old value: channels with indices above new config value are unregistered.
    New config value is smaller than old value: when channels with all indices from old config value are registered, then new channels with indices up to the new value are registered.
    Property AllowAutoRegistration must be also True for this feature to work in this method

Initialized to False.
